     3 ОПИС ВИКОНАНОЇ РОБОТИ
     3.1 Загальні відомості про шаблони (патерни) проєктування
     Шаблони проєктування - це перевірені часом типові способи вирішення поширених проблем, що виникають під час проектування програмного забезпечення. Згідно з класифікацією банди чотирьох (GoF), патерни поділяються на три групи:
* породжувальні: відповідають за механізми створення об'єктівж
* структурні: визначають, як об'єкти та класи збираються у великі структури;
* поведінкові: відповідають за ефективну комунікацію між об'єктами.

3.2  Опис обраного патерна
   
     Назва патерна: Decorator (Декоратор).
     Тип шаблона: Структурний.
     Основне призначення: Додавання нової поведінки або обов'язків об'єкту динамічно, без зміни його класу. Це дозволяє розширювати функціональність об'єктів на льоту.
     Передумови застосування: Використання є доцільним, коли необхідно додати обов'язки окремим об'єктам, а не всьому класу, або коли розширення за допомогою спадкування стає неможливим через вибухове зростання кількості підкласів.
     Переваги:
* висока гнучкість у порівнянні зі статичним спадкуванням;
* можливість динамічного розширення функціональності;
* дотримання принципу відкритості/закритості (Open/Closed Principle);
* зменшення кількості підкласів у системі.
Недоліки:
* поява великої кількості маленьких об'єктів (класів декораторів);
* ускладнення процесу конфігурації об'єктів та читання коду;
* важче відстежувати порядок обгортання об'єктів.
     
     3.3 Структура шаблона (патерна)
     
     Структура патерна базується на композиції об'єктів:
* Component (Компонент): інтерфейс або абстрактний клас, що визначає загальну поведінку для базового об'єкта та його декораторів;
* ConcreteComponent (Конкретний компонент): об'єкт, до якого додається нова поведінка;
* Decorator (Декоратор): абстрактний клас, що містить посилання на об'єкт компонента та реалізує його інтерфейс;
* ConcreteDecorator (Конкретний декоратор): додає нові стани або поведінку компоненту.
     Опис взаємодії: Декоратор обгортає базовий об'єкт і делегує йому виконання основних методів, додаючи свою логіку до або після виклику основного методу.
     На рисунку 3.1 можна побачити UML-діаграму на основі прикладу коду, що було написано для даної доповіді.
     Опис структурних частин:
* ICoffee (Component): інтерфейс, що визначає загальний набір методів (GetDescription, GetCost) як для базових компонентів, так і для їхніх декораторів;
* SimpleCoffee (ConcreteComponent): конкретний клас, що реалізує базову функціональність об'єкта, до якого можна додавати нові обов'язки;
* CoffeeDecorator (Decorator): абстрактний клас, який зберігає посилання на об’єкт типу ICoffee та реалізує той самий інтерфейс, делегуючи йому запити;
* MilkDecorator / ChocolateDecorator (ConcreteDecorator): конкретні класи-декоратори, що додають нову поведінку (інгредієнти та їхню вартість) базовому об'єкту динамічно.

     3.4 Реалізація шаблона (патерна)
     
     Реалізація виконана мовою програмування C#. Програмний код можна знайти у Додатку В.

ДОДАТОК В
Приклади програмного коду
	Промпт для штучного інтелекту:
     Напиши програму мовою C#, яка демонструє роботу структурного патерна проєктування Decorator на прикладі замовлення кави. Створи інтерфейс ICoffee з методами string GetDescription() та double GetCost(). Реалізуй базовий клас SimpleCoffee, де опис — "Simple coffee", а ціна — 5.0. Створи абстрактний клас CoffeeDecorator, який реалізує ICoffee, містить захищене поле _coffee та ініціалізує його через конструктор. Методи мають бути віртуальними та викликати методи вкладеного об'єкта. Реалізуй два конкретні декоратори. MilkDecorator: додає до опису ", milk" та 1.5 до вартості. ChocolateDecorator: додає до опису ", chocolate" та 2.0 до вартості. У методі Main продемонструй створення об'єкта SimpleCoffee, який послідовно обгортається в обидва декоратори. Виведи фінальний опис та загальну вартість у консоль.
     Код, який можна отримати в результаті:
1. internal class Program
2. {
3.     public interface ICoffee
4.     {
5.         string GetDescription();
6.         double GetCost();
7.     }
8. 
9.     public class SimpleCoffee : ICoffee
10.     {
11.         public string GetDescription()
12.         {
13.             return "Simple coffee";
14.         }
15. 
16.         public double GetCost()
17.         {
18.             return 5.0;
19.         }
20.     }
21. 
22.     public abstract class CoffeeDecorator : ICoffee
23.     {
24.         protected ICoffee _coffee;
25. 
26.         public CoffeeDecorator(ICoffee coffee)
27.         {
28.             _coffee = coffee;
29.         }
30. 
31.         public virtual string GetDescription()
32.         {
33.             return _coffee.GetDescription();
34.         }
35. 
36.         public virtual double GetCost()
37.         {
38.             return _coffee.GetCost();
39.         }
40.     }
41. 
42.     public class MilkDecorator : CoffeeDecorator
43.     {
44.         public MilkDecorator(ICoffee coffee) : base(coffee) { }
45. 
46.         public override string GetDescription()
47.         {
48.             return _coffee.GetDescription() + ", milk";
49.         }
50. 
51.         public override double GetCost()
52.         {
53.             return _coffee.GetCost() + 1.5;
54.         }
55.     }
56. 
57.     public class ChocolateDecorator : CoffeeDecorator
58.     {
59.         public ChocolateDecorator(ICoffee coffee) : base(coffee) { }
60. 
61.         public override string GetDescription()
62.         {
63.             return _coffee.GetDescription() + ", chocolate";
64.         }
65. 
66.         public override double GetCost()
67.         {
68.             return _coffee.GetCost() + 2.0;
69.         }
70.     }
71. 
72. 
73.     static void Main(string[] args)
74.     {
75.         ICoffee coffee = new SimpleCoffee();
76.         coffee = new MilkDecorator(coffee);
77.         coffee = new ChocolateDecorator(coffee);
78. 
79.         Console.WriteLine(coffee.GetDescription());
80.         Console.WriteLine("Total cost: " + coffee.GetCost());
81.     }
82. }
