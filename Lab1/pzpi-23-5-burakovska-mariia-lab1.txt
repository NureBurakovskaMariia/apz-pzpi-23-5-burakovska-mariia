     3 ОПИС ВИКОНАНОЇ РОБОТИ

     Система являє собою єдиний цифровий простір для автоматизації управління притулками для тварин. Вона забезпечує облік тварин (медична історія, статус усиновлення), координацію волонтерів, прозорий збір пожертв та моніторинг умов утримання (температура, вологість) за допомогою ІоТ-модулів . Система орієнтована на адміністрацію притулків, волонтерів та небайдужих громадян .
	У ході виконання лабораторної роботи було проведено доопрацювання документа Vision & Scope для системи допомоги тваринам у притулках. Основна увага була зосереджена на деталізації меж проєкту та плануванні впровадження інтелектуальних модулів.
     Було виконано такі кроки:
* Визначення меж першого релізу (Initial Release): Описано взаємодію серверної частини на Node.js, веб-інтерфейсу на React та мобільного клієнта на Kotlin з урахуванням нових можливостей бази даних.
* Проєктування інтеграції ШІ: Розроблено стратегію впровадження штучного інтелекту для майбутніх ітерацій, зокрема для автоматичного підбору тварин та аналізу стану здоров'я через ІоТ-датчики.
* Аналіз обмежень: Визначено технічні та функціональні ліміти системи, включаючи питання апаратної залежності ІоТ-модулів та межі автоматизації медичних процесів.
      Повний опис оновлених розділів Vision & Scope наведено у Додатку Б.

ДОДАТОК Б
     1.1. Аналіз проблеми
   У сучасних притулках для тварин часто виникають труднощі з ефективним управлінням ресурсами, контролем умов утримання, обліком тварин і координацією волонтерів.
   Багато процесів виконуються вручну або ведуться у таблицях, що призводить до:
* втрати або дублювання інформації про тварин;
* відсутності системного моніторингу стану притулку;
* труднощів у комунікації між адміністрацією та волонтерами;
* низької прозорості у звітності щодо донатів.
   Це знижує ефективність роботи та негативно впливає на якість догляду за тваринами.
Тому виникає потреба у комплексній програмній системі, яка дозволить автоматизувати основні процеси, забезпечити аналітику та контроль умов утримання.
     1.2. Аналіз аналогів
   	Існуючі рішення (наприклад, ShelterBuddy, PetPoint, RescueGroups) надають окремі функції — облік тварин або управління волонтерами — але не підтримують моніторинг середовища та інтеграцію з IoT.
   	Основні недоліки аналогів:
* відсутність інтеграції між обліком тварин і умовами утримання;
* складні інтерфейси або висока вартість;
* обмежений доступ через мобільні пристрої.
   	Запропонована система покращить ситуацію за рахунок:
* єдиної бази даних для адміністраторів, волонтерів і користувачів;
* веб- та мобільного клієнтів з інтуїтивним інтерфейсом;
* модулю-емулятора IoT, який генерує тестові дані про температуру й вологість у приміщенні притулку, імітуючи роботу сенсорів.
2.1. Опис призначення системи
	Система призначена для:
* обліку тварин (дані, медична історія, вакцинації, статус усиновлення);
* управління волонтерами, завданнями та графіками;
* ведення обліку донатів та матеріальної допомоги;
* моніторингу умов утримання тварин через модуль-емулятор IoT;
* взаємодії з користувачами, які бажають усиновити тварину.
	Цінність для користувачів:
* для адміністрації — автоматизація обліку й аналітики;
* для волонтерів — зручне планування роботи;
* для громадян — швидкий доступ до інформації про тварин і можливість допомоги притулку.
     2.2 Опис бізнес-логіки
Основні бізнес-процеси системи:
* адміністратор або працівник притулку додає інформацію про нову тварину;
* система зберігає дані у центральній базі;
* IoT-емулятор передає тестові показники середовища до сервера;
* сервер обробляє дані, формує записи у базі та, при потребі, генерує сповіщення (наприклад, про підвищену температуру в клітці);
* через веб-або мобільний застосунок користувачі зможуть переглядати інформацію, редагувати її, отримувати повідомлення та статистику.
3.1 Scope of Initial Release
     Початковий реліз системи охоплює повний цикл взаємодії між усіма компонентами архітектури:
* Серверна частина (Node.js + Express): Реалізація REST API для обробки запитів від усіх клієнтів, керування базою даних SQLite та авторизація користувачів.
* Веб-клієнт (React): Повнофункціональна панель адміністратора для ведення обліку тварин, візуалізації даних з датчиків ІоТ у вигляді графіків та управління волонтерськими завданнями .
* Мобільний клієнт (Kotlin): Застосунок для користувачів та волонтерів, що дозволяє переглядати базу тварин, отримувати push-сповіщення про термінові потреби притулку та здійснювати швидкі донати .
* ІоТ-компонент (Python-скрипти): Робота сенсорів, що збирають дані про температуру та вологість у вольєрах і передають їх на сервер для подальшого аналізу.
   3.2 Scope of Subsequent Releases (Інтеграція ШІ)
     В наступних версіях системи планується впровадження штучного інтелекту для покращення користувацького досвіду та операційної ефективності:
* Застосування: 
1.  Smart Matching: Система рекомендацій на основі ШІ для підбору ідеальної тварини для потенційного власника (враховуючи спосіб життя людини та темперамент тварини).
2. Health Prediction: Аналіз даних з ІоТ-датчиків для виявлення аномалій у поведінці або середовищі, що можуть свідчити про початок хвороби тварини.
3. Visual Recognition: Автоматичне тегування характеристик тварин (порода, колір, приблизний вік) за завантаженим фото.
* Збирання даних: Дані збиратимуться через анкети усиновлення, історію медичних записів, логи активності волонтерів та архівні показники ІоТ-датчиків. Фотографії тварин, завантажені волонтерами, стануть базою для навчання візуальних моделей.
* Навчання/Використання: Для аналізу зображень та тексту доцільно використовувати готові API (наприклад, Google Cloud Vision або OpenAI API). Для системи рекомендацій планується донавчання існуючих моделей машинного навчання на специфічних даних притулку для досягнення вищої точності персоналізації.
     3.3 Limitations and Exclusions
* Залежність від мережі: Функціональність моніторингу ІоТ критично залежить від стабільного інтернет-з’єднання; при його відсутності дані можуть втрачатися.
*  Виключений функціонал: Прямі медичні консультації: Система свідомо не надає автоматичних діагнозів або порад щодо лікування, щоб уникнути юридичної відповідальності та шкоди тваринам (тільки фіксація даних від ветеринарів). Маркетплейс товарів: Продаж товарів для тварин виключений з початкового скоупу, щоб сфокусуватися на благодійності та волонтерстві, не ускладнюючи податкову звітність системи.
